**Option A — React component inside the Signalum codebase** is better long-term, for these reasons:

**Easier to modify:** When you want to update copy, add a section, or change the CTA, you just edit the file in Replit and it deploys with the rest of the app. No separate file to track down.

**Shared design system:** The landing page can import shadcn/ui components, Tailwind classes, and any shared utilities already in the codebase. If you ever want to pull in a real live stat from the API (e.g. "X signals tracked today"), it's trivial.

**One deployment:** Single `npm run build` → Republish covers everything. With a standalone HTML file you'd need to manage it separately.

**Route control:** The React router already handles unauthenticated vs authenticated routing. The landing page becomes just another route — `/` shows the landing page if not logged in, redirects to `/` (app) if logged in.

**The only downside:** Converting the HTML to a React component takes a bit more work now. But it's a one-time cost and pays off every time you edit it after.

> **Conversion rules:**
> - `class=` → `className=`
> - All `<style>` block CSS → move into a `<style>` tag injected via a `useEffect` that appends it to `document.head`, OR keep it as a plain `<style>` tag inside the component return (Vite supports this)
> - The Google Fonts `<link>` tag → add to `client/index.html` `<head>` section
> - `onclick="showTab(0)"` → convert to React `onClick` handlers using `useState` for active tab
> - The scroll reveal `IntersectionObserver` → put in a `useEffect`
> - The CTA form submit handler → `onClick` on the button with `useState` for email value
> - Import `{ useState, useEffect }` from react
> - Default export the component
>
> **Register the route:**
> In the main router file (likely `client/src/App.tsx`), find where routes are defined. Add:

> - The existing unauthenticated redirect logic may already handle this — check before adding duplicate logic
>
> **The HTML to convert** is the full content of the landing page file. Here is the complete HTML:

>
> **Critical rules:**

> - No drizzle-kit push
> - Keep all existing authenticated routes working exactly as before
