## What is Signalum?

Signalum is a **multi-tenant competitive intelligence SaaS platform** built by Rohin (Product Manager at Entrust). Live at `signalum.rohin.co`. Tracks competitors, regulations, and industry topics using Perplexity for ambient intelligence and Claude for AI analysis.

Rohin is **non-technical** — Claude diagnoses issues and writes precise prompts for **Replit AI** to execute.

## Critical Rules (ALWAYS follow)

1. **NEVER run `drizzle-kit push`** — drops ~25 columns
2. **Use `db.execute(sql\`...\`)` for all DB writes** — never `pool.query`
3. **New DB columns/tables** → add to `server/dbSafety.ts` only
4. **DB connection** → `DATABASE_URL` with `?ssl` suffix
5. **Never "Copy dev to production"** in Replit — wipes data
6. **Deploy** → source edit → `npm run build` → Republish
7. **`staleTime: Infinity` globally** — add `staleTime: 0, gcTime: 0, refetchOnMount: "always"` to queries needing fresh data
8. **Shell DB ≠ production DB** — always verify via browser/API
9. **No JS arrays in Drizzle sql** — use `sql.join(ids.map(id => sql\`${id}::uuid\`), sql\`, \`)` for IN clauses
10. **Use `apiRequest()` for authenticated fetches** — raw `fetch()` has no JWT
11. **AI generation = on demand only** — store in DB, serve from cache, regenerate via Refresh button (never on page load)
12. **Strategic Pulse = SINGLE Claude API call** — multiple sequential calls hit 30k TPM limit
13. **Manual overrides persist** — rows with `source='manual'` are never overwritten by Perplexity
14. **Never define form components inside parent components**

## Database Tables

### Core
- **`workspaces`** — one per user, `categories` JSONB with all entities, many profile columns
- **`user_profiles`** — role, onboarding, digest settings
- **`captures`** — intel items matched to entities (has `focus_area`, `extracted_excerpt`, `suggested_new_category`)

### Competitive Dimensions
- **`competitive_dimensions`** — dimension groups per workspace: `name`, `source` (universal/ai/custom), `priority`, `items` (jsonb: `[{name, our_status, notes}]`)
- **`competitor_dimension_status`** — per-competitor per-item status: `status` (yes/partial/no/unknown), `source` (perplexity/manual), `evidence`

### Topic Intelligence
- **`topic_notes`** — rich text notes per topic (HTML), UNIQUE(workspace_id, entity_id)
- **`topic_milestones`** — timeline entries: `date`, `event_text`, `source` (manual/perplexity), dedup index
- **`topic_impact_analysis`** — AI-generated: `relevance` (high/medium/low), `relevance_reason`, `insights` (jsonb), `actions` (jsonb), UNIQUE(workspace_id, entity_name)
- **`entity_tracking_intent`** — user's tracking focus: `selected_focuses` (text[]), `custom_focus`, UNIQUE(workspace_id, entity_name)

### Entity Data
- `entity_products`, `entity_geo_presence`, `entity_funding`, `entity_partnerships`, `entity_certifications`, `entity_capabilities`, `entity_intelligence`, `entity_win_loss`, `entity_swot`, `entity_products_managed`

- `product_context` — product description for AI prompts
- `competitor_pricing` — pricing data

### Infrastructure

## Feature Architecture

### Competitive Dimensions
**One source of truth, three views:**
1. **My Product settings** → define dimensions, items, our_status, priorities
2. **Competitor page** → DimensionComparisonCard (Us vs Them) + DimensionSpiderChart (radar + scoring)
3. **My Workspace Compare** → DimensionMatrix (all competitors × all dimensions)

**Scoring** (DimensionSpiderChart): yes=100, partial=50, unknown=25, no=0 averaged per dimension. Badges: Ahead (>10pts), Parity (±10), Behind (<-10). Overall = average of dimension scores.

**Perplexity research**: `researchEntityDimensions()` in ambientSearch.ts, 7-day staleness, skips `source='manual'` rows.

### Tracking Intent
1. User adds/views topic → TrackingFocusStep shows AI-suggested focus areas
2. User selects → saved to `entity_tracking_intent`
3. Drives: AI Summary generation, Impact analysis, capture `focus_area` tagging

### Topic Impact Analysis (non-competitor topics)
- AI-generated via Claude using tracking focus + dimensions + recent captures
- Structured output: relevance level, insights (risk/warning/strength), recommended actions
- Generated on demand (not page load), stored in `topic_impact_analysis`
- Truncated JSON recovery logic (regex match + bracket closing)

### Non-Competitor Topic Pages
**Tabs:** Overview + Updates only
**Sections** (all CollapsibleSection):
1. AI Summary — on demand, stored in DB, Refresh button
2. Impact on our product — AI-generated from tracking focus + dimensions
3. My Notes — rich text editor (bold, italic, lists, heading, link)
4. Key Milestones — timeline (manual + Perplexity auto, 30-day staleness)
5. Latest Updates — compact cards from UpdatesList

**Removed:** Partnerships, Key Dates (Coming Soon), Compliance Notes (Coming Soon)

### Competitor Pages
**Tabs:** Overview, Profile, Commercial, Competitive, Strategy, Updates
**All sections CollapsibleSection.** Key sections:
- AI Summary + So What Card
- Competitive Scoring (DimensionSpiderChart) — radar + per-dimension scores
- Competitive Dimensions (DimensionComparisonCard) — side-by-side comparison
- Geographic Presence (GeoPresenceMap) — D3 world map + region cards
- Products, Funding, Partnerships, Battlecard, Win/Loss, SWOT
- Updates (UpdatesList)
- **DimensionMatrix NOT on competitor pages** — only in My Workspace Compare

### Updates UI (UpdatesList)
- **Filter bar:** time pills + category dropdown
- **Compact cards:** signal dot (red/amber/blue) + title + date + source + dimension pill + category pill
- **Expand on click:** full text + action buttons
- **Category detection:** client-side keyword regex on title
- **Dimension tags:** client-side match against dimension item names
- **Pricing:** only for competitors, only real product pricing (not revenue/funding)
- **Capture relevance:** entity name validated in content before saving (competitors only)

### Strategic Pulse (7 sections)
1. Market Direction — 2-3 paragraph outlook
2. Market Forces — structural forces
3. Emerging Opportunities
4. Competitor Moves Decoded
5. Threat Radar & Watch List — urgent + monitoring tiers
6. Regional Intelligence — 2×3 grid
7. Roadmap Implications — recommendations linking signals to dimensions

**Single Claude API call**, max_tokens 4096, dimension context included.
**Known issue:** first 5 sections may not populate — verify single-call code is deployed.

### Ambient Search Cron (6AM UTC)
- Per competitor: `researchEntity` (30-day) + `researchEntityDimensions` (7-day)
- Per non-competitor: auto-milestones (30-day)
- Capture relevance: validates entity name in content before saving
- Rate limiter: 10 calls/minute
- ~42 entities across 5 categories
