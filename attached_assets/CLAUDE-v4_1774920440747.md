## What is Signalum?

Signalum is a **multi-tenant competitive intelligence SaaS platform** built by Rohin (Product Manager at Entrust). It is live at `signalum.rohin.co` (also accessible at `watchloom.rohin.co`). The app tracks competitors, regulations, and industry topics for a given workspace, using Perplexity for ambient intelligence and Claude for AI-generated summaries and strategic analysis.

Rohin is **non-technical** and works by having Claude diagnose issues and write precise prompts for **Replit AI** to execute.

## Critical Infrastructure Rules

1. **NEVER run `drizzle-kit push`** — will drop ~25 columns
2. **Always use `db.execute(sql\`...\`)` NOT `pool.query`** for INSERT/UPDATE in new code
3. **New DB columns** → always add to `server/dbSafety.ts` with `ADD COLUMN IF NOT EXISTS`
4. **DB connection** → `DATABASE_URL` with `?ssl` suffix, unified dev+prod via Replit Secrets
5. **Never use "Copy dev to production"** in Replit — wipes production data
6. **Before any deploy** → check migration preview for DROP statements, cancel if found
7. **Replit deployment fix** → source edit → `npm run build` → Republish (only reliable path)
8. **`staleTime: Infinity`** is set globally in `queryClient.ts` — always add `staleTime: 0, gcTime: 0, refetchOnMount: "always"` to individual queries that need fresh data
9. **Replit shell vs production server may use different DB connections** — always verify data through the browser/API, not just shell queries
10. **Drizzle `sql` template cannot pass JS arrays** — use `sql.join(ids.map(id => sql\`${id}::uuid\`), sql\`, \`)` for IN clauses
11. **Always use `apiRequest()` for authenticated frontend fetches** — raw `fetch()` won't include the Supabase JWT token
12. **AI summaries and impact analyses must be generated on demand, not on page load** — store in DB, serve from cache, only regenerate via manual "Refresh" button or when tracking focus is set
13. **Strategic Pulse must use a SINGLE Claude API call** — multiple sequential calls hit the 30k TPM rate limit and fail silently

## Database Schema Summary

### Core tables
- **`workspaces`** — one per user, contains `categories` JSONB column with all entities + many profile columns
- **`user_profiles`** — role, onboarding context, digest settings
- **`captures`** — intel items matched to entities (has `focus_area` column for tracking intent)

### Competitive Dimensions (March 2026)
- **`competitive_dimensions`** — dimension groups per workspace
  - `id`, `workspace_id`, `name`, `source` (universal/ai/custom), `priority` (high/medium/low), `display_order`, `items` (jsonb: `[{name, our_status, notes}]`), `created_at`, `updated_at`
- **`competitor_dimension_status`** — per-competitor status per dimension item
  - `id`, `dimension_id`, `entity_name`, `item_name`, `status` (yes/partial/no/unknown), `source` (perplexity/manual), `evidence`, `last_updated`
  - **Manual overrides** (`source='manual'`) are never overwritten by Perplexity

### Topic Intelligence (March 2026)
- **`topic_notes`** — rich text user notes per topic
  - `id`, `workspace_id`, `entity_id`, `content` (HTML), `updated_at`
  - UNIQUE(workspace_id, entity_id)
- **`topic_milestones`** — timeline milestones per topic (manual + auto)
  - `id`, `workspace_id`, `entity_id`, `date`, `event_text`, `source` (manual/perplexity), `created_at`
  - Dedup index on (workspace_id, entity_id, date, event_text)
- **`topic_impact_analysis`** — AI-generated impact analysis per topic
  - `id`, `workspace_id`, `entity_name`, `relevance` (high/medium/low), `relevance_reason`, `insights` (jsonb), `actions` (jsonb), `generated_at`
  - UNIQUE(workspace_id, entity_name)
- **`entity_tracking_intent`** — user's tracking focus areas per entity
  - `id`, `workspace_id`, `entity_name`, `selected_focuses` (text[]), `custom_focus`, `updated_at`
  - UNIQUE(workspace_id, entity_name)

### Entity data tables
- `entity_products` — manually added products (checked FIRST before JSON fallback)
- `entity_geo_presence` — geographic presence data
- `entity_funding` — funding data
- `entity_partnerships` — partnerships
- `entity_certifications` — certifications
- `entity_capabilities` — per-entity capabilities
- `entity_intelligence` — AI-generated intelligence fields
- `entity_win_loss` — win/loss records
- `entity_swot` — SWOT analysis
- `entity_products_managed` — tracks entities with manually managed products
- `competitor_pricing` — pricing data
- `battlecards` — AI-generated battlecards
- `strategic_directions` — where heading / what means for us
- `strategic_pulse` — NOT managed by Drizzle migrations

### Legacy capability tables (DEPRECATED)
- `workspace_capabilities`, `competitor_capabilities`, `entrust_capabilities`, `our_product_capabilities`
- **DO NOT USE in new code.** All new capability logic uses `competitive_dimensions` and `competitor_dimension_status`.

### Other infrastructure tables
- `workspace_context` — domain keywords for tenant
- `topic_dates` — key dates per entity
- `monitored_urls` — URLs to monitor per entity
- `product_context` — product description context for AI prompts

### Key workspace fields (flat columns)
`user_perspective`, `org_description`, `org_geographies`, `tracking_types`, `competitors`, `win_factors`, `vulnerability`, `early_warning_signal`, `display_name`, `parent_workspace_id`, `onboarding_completed`, `briefing_enabled`, `briefing_day`, `briefing_time`, `capture_token`

## Competitive Dimensions System

### Architecture: One source of truth, three views
1. **Settings (My Product page)** — user defines dimensions, sets items, sets OUR status, sets priorities
2. **Competitor page (DimensionComparisonCard)** — side-by-side Us vs Competitor with vulnerability alerts
3. **Capability Matrix (DimensionMatrix)** — bird's-eye filterable grid (accessible from Compare button on My Workspace)

### Dimension sources
- **Universal** — apply to any market
- **AI-suggested** — Claude generates based on product description
- **Custom** — user adds manually

### How dimensions feed Perplexity search
`researchEntityDimensions` in `ambientSearch.ts`:
- Groups items by dimension with priority labels
- Includes entity disambiguation context + category focus + product context
- Asks Perplexity for `{status, evidence}` per item
- Skips rows where `source='manual'`

### Dimension scoring (DimensionSpiderChart)
- Scores per dimension: yes=100, partial=50, unknown=25, no=0 averaged across items
- Radar chart overlays Us (purple #534AB7) vs competitor (coral #D85A30)
- Overall competitive score = average of dimension scores
- Assessment badges: Ahead (>10 pts), Parity (within 10), Behind (<-10 pts)

### API Routes for Dimensions
| Method | Route | Purpose |
|---|---|---|
| GET | `/api/dimensions` | All dimensions for workspace |
| POST | `/api/dimensions` | Create dimension |
| PUT | `/api/dimensions/:id` | Update dimension |
| DELETE | `/api/dimensions/:id` | Delete dimension + statuses |
| POST | `/api/dimensions/suggest` | AI-suggest via Claude |
| GET | `/api/competitor-dimensions/:entityName` | Comparison data |
| PUT | `/api/competitor-dimension-status/:id` | Manual override |
| GET | `/api/matrix/dimensions` | Full matrix data (uses `sql.join`) |
| POST | `/api/search/research-dimensions/:entityName` | Manual research trigger |

## Tracking Intent System (March 2026)

When a user adds a topic or views one for the first time, they're prompted with "What do you most want to track?" — AI suggests focus areas, user selects. This drives:
- **AI Summary generation** — focused on selected areas
- **Impact analysis** — personalized to user's focus + competitive dimensions
- **Perplexity ambient search** — captures tagged with `focus_area`

### Flow:
1. User adds/views topic → TrackingFocusStep shows AI-suggested focuses
2. User selects focuses → saved to `entity_tracking_intent` table
3. AI Summary generated using focus areas as prompt context
4. Impact analysis generated referencing focus + dimensions + recent captures
5. Captures from ambient search tagged with matching `focus_area`

## Ambient Search / Perplexity Cron

- **Schedule**: 6AM UTC daily
- **Trigger manually**: `curl -X POST http://localhost:5000/api/search/run-ambient`
- **Per competitor**: `researchEntity` (funding/geo/products, 30-day staleness) + `researchEntityDimensions` (7-day staleness)

- **Rate limiter**: 10 calls per minute
- **JSON parsing**: strips markdown code fences before `JSON.parse`
- **Entities**: ~42 total across 5 categories

## Known Patterns & Gotchas

### Always do this
```typescript
await db.execute(sql`...`);  // for INSERT/UPDATE
useQuery({ queryKey: [...], staleTime: 0, gcTime: 0, refetchOnMount: "always" })
const res = await apiRequest("GET", "/api/...");  // authenticated fetch
WHERE id IN (${sql.join(ids.map(id => sql`${id}::uuid`), sql`, `)})  // IN clause
if (existingRow.source === 'manual') { /* skip Perplexity update */ }
```

### Never do this
```typescript
pool.query('UPDATE ...')  // ❌ use db.execute
fetch("/api/...").then(r => r.json())  // ❌ no auth, use apiRequest
WHERE id = ANY(${jsArray}::uuid[])  // ❌ Drizzle can't handle arrays
drizzle-kit push  // ❌ drops columns
function Parent() { function Child() {} }  // ❌ define forms separately
// Calling Claude API on every page load  // ❌ generate once, store, serve from DB
```
