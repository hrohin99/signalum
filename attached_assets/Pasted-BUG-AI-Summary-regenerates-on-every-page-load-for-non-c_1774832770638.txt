BUG: AI Summary regenerates on every page load for non-competitor 
topics. This is expensive and unnecessary.

CURRENT BEHAVIOR: Every time the topic page loads, it calls the 
Claude API to generate a fresh AI summary. This wastes API tokens 
and slows down page load.

DESIRED BEHAVIOR: 
1. AI Summary is generated ONCE and stored in the database
2. On page load, fetch the stored summary (fast, no API call)
3. Only regenerate when the user clicks a "Refresh" button
4. Auto-generate when tracking focus is first set (same as impact)
