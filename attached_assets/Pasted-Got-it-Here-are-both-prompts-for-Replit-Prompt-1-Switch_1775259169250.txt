**Prompt 2 — Heatmap cell drill-down side panel**

> **Task: Add drill-down side panel when clicking a heatmap cell in Market Signals**
>
> In `client/src/pages/market-signals.tsx`, add a side panel that slides in from the right when a heatmap cell is clicked.
>
> **Trigger:**
> When user clicks a heatmap cell with count > 0, open the side panel with the selected `dimension_name` and `item_name` as context. Clicking a cell with count 0 does nothing.
>
> **New endpoint in `server/routes.ts`:**
>
> ```
> GET /api/market-signals/item-signals?dimension_name=X&item_name=Y
> requireAuth
> ```
>
> Returns all signals that have a requirement linked to this dimension item:
> ```typescript
> const result = await db.execute(sql`
>   SELECT
>     ms.id AS signal_id,
>     ms.name AS signal_name,
>     ms.source_type,
>     ms.source_organisation,
>     ms.signal_date,
>     ms.status,
>     msr.id AS requirement_id,
>     msr.requirement_text,
>     msr.source_reference
>   FROM requirement_dimension_links rdl
>   JOIN market_signal_requirements msr ON msr.id = rdl.requirement_id
>   JOIN market_signals ms ON ms.id = msr.signal_id
>   WHERE rdl.workspace_id = ${workspaceId}
>   AND rdl.dimension_name = ${req.query.dimension_name}
>   AND rdl.item_name = ${req.query.item_name}
>   ORDER BY ms.created_at DESC
> `);
> return res.json({ results: result.rows });
> ```
>
> **Side panel UI:**
>
> - Fixed width: 380px, slides in from right, page content does not shift (use absolute/overlay positioning)
> - Backdrop: semi-transparent overlay behind panel, click backdrop to close
> - Close button (X) top right of panel
>
> **Panel header:**
> - Item name as title (font-size 15px, font-weight 500)
> - Dimension name below in muted text
> - Count badge: "N signals" in purple pill
>
> **Panel body — one card per signal:**
> ```
> [Source type badge] [Status badge]
> Signal name (font-weight 500)
> Source org · Date
>
> Requirement:
> "[requirement_text in italics]"
> Source reference in muted text below
> ```
>
> Each signal card has a subtle border, `border-radius: var(--border-radius-lg)`, padding 12px. Source type badge uses same colour mapping as the signals list.
>
> If multiple requirements from the same signal reference this item (edge case), show them stacked within the same signal card.
>
> **Loading state:** show a spinner inside the panel while fetching.
>
> **Empty state:** "No signals reference this item yet." (shown for 0-count cells if somehow opened — though clicking 0 cells is blocked on the frontend).
>
> **Critical rules:**
> - Never run `drizzle-kit push`
